## Correction

Three of my early memos to you (and to PO before that) cited the cross-pollination brief landing path as `docs/briefs/cross-pollination/current.md`. **For OpenLaws specifically, that path is wrong.**

OpenLaws uses a different convention from the other six readers in the cross-pollination network:

| Reader | Path convention |
|--------|----------------|
| Klatch | `docs/briefs/cross-pollination/YYYY-MM-DD.md` + overwrite `current.md` |
| Piper Morgan | `docs/briefs/cross-pollination/YYYY-MM-DD.md` + overwrite `current.md` |
| Dispatch | `intelligence/cross-pollination/YYYY-MM-DD.md` + overwrite `current.md` |
| Rebel Alliance | `docs/briefs/cross-pollination/YYYY-MM-DD.md` + overwrite `current.md` |
| Weather | `docs/briefs/cross-pollination/YYYY-MM-DD.md` + overwrite `current.md` |
| NYT-Crossword | `docs/briefs/cross-pollination/YYYY-MM-DD.md` + overwrite `current.md` |
| **OpenLaws** | **`dispatch/briefing-YYYY-MM-DD.md`** — one file per day, **no `current.md` pointer** |

So when reading or referencing the OpenLaws brief landing point, use `~/cool/OpenLaws/dispatch/briefing-YYYY-MM-DD.md` (and look for the most recent date), not `docs/briefs/cross-pollination/current.md`.

The general `current.md` pattern remains correct for the other six readers — this correction is OpenLaws-specific.
