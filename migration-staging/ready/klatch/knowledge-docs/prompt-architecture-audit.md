# Prompt Architecture Audit — March 16, 2026

Captured from PO + Daedalus design session. Intended as input for the domain model sketch and subsequent implementation.

## Current 4-Layer Prompt Assembly

When a message is sent in Klatch, `buildSystemPrompt()` concatenates these layers (separated by `\n\n`):

1. **Kit briefing** — Orientation text for imported channels ("you're in Klatch, no tools"). Includes MEMORY.md fallback for channels without a project link. Skipped for native channels.
2. **Project instructions** — From `projects.instructions` (populated from CLAUDE.md or prompt_template at import, or user-written).
3. **Channel addendum** — From `channels.system_prompt` field. Always empty for imports. Only meaningful for klatches (room topic). Vestigial for chats.
4. **Entity system prompt** — From `entities.system_prompt`. The persona/role definition. Currently "You are a helpful assistant." for all imported chats (default entity).

## Decisions (March 16)

1. **MEMORY.md moves to project level.** Add `memory` column to `projects` table. Stop storing in per-channel sourceMetadata. Build a minimal project settings view (name, instructions, memory — all editable).

2. **Don't drop claude.ai memories.** Normalize and store them into the project `memory` field alongside MEMORY.md content. Both serve the same purpose (accumulated knowledge).

3. **Role/persona is a future opportunity.** Imported chats get default entity; users customize manually for now. Architecture should leave room for entity-from-import later.

4. **Channel addendum hidden for chats.** Field exists in schema (null), but chat settings UI doesn't show it. Only klatch settings surface it as "room topic."

## Gaps Identified

- **MEMORY.md is homeless** — stored per-channel but is a project-level asset. Same memory duplicated across N imports from same project.
- **claude.ai memories are dropped** — parsed in preview but never stored or injected.
- **No role/persona imports** — neither source includes extractable persona definitions.
- **Kit briefing is invisible** — system prompt injection shapes behavior but creates no observable moment for user or agent. See issue #13.
- **Admin UI is misleading** — shows only layer 3 (empty for imports), not the assembled prompt. See issue #9.
