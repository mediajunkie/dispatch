## Quick Reference Scoring Guide

### Completeness Scale (1-5)
1. **Minimal**: Addresses only part of the problem
2. **Basic**: Covers main points but misses important details
3. **Good**: Comprehensive coverage of the scenario
4. **Excellent**: Thorough analysis plus valuable insights
5. **Outstanding**: Everything above plus proactive suggestions

### PM Judgment Scale (1-5)
1. **Poor**: Misunderstands PM priorities or stakeholder needs
2. **Weak**: Correct but generic advice, no PM sophistication
3. **Good**: Shows PM understanding, appropriate recommendations
4. **Strong**: Demonstrates advanced PM thinking and trade-off analysis
5. **Expert**: Insights you wish you'd thought of, sophisticated stakeholder navigation

### Cognitive Load Scale (1-5)
1. **Effortless**: Can immediately act on the advice
2. **Easy**: Minor additional thinking required
3. **Moderate**: Need to adapt advice to specific situation
4. **Heavy**: Requires significant interpretation or additional work
5. **Overwhelming**: Too complex or confusing to be useful

## Statistical Tracking

### Sample Size Goals
- **Phase 1**: 10 tests across 3 scenario types
- **Phase 2**: 25 tests with refined scenarios
- **Phase 3**: 50+ tests including real-work scenarios

### Confidence Intervals
Track when you feel confident about patterns:
- **Suggestive**: 3-5 tests showing same pattern
- **Likely**: 5-10 tests confirming pattern
- **Confident**: 10+ tests with statistical significance

### Bias Mitigation
- Rotate which system goes first
- Use predetermined scenarios (don't adapt in real-time)
- Wait at least 1 day between testing same scenario
- Document your mood/context for each test session
