### Learning pattern structure

The learning pattern section is what makes Weekly Ships valuable beyond status updates. It captures **transferable insights** that can help other teams/projects.

**Required components** (all 5):
1. **Discovery**: One-sentence description of what was learned
2. **Example from this week**: Specific instance showing the pattern
3. **Why it matters**: Impact and what it addresses
4. **Application beyond this week**: How to apply elsewhere
5. **Related patterns**: Links to similar patterns

**Good learning patterns**:
- Specific and concrete (not abstract principles)
- Include real examples from the week
- Show cause and effect clearly
- Applicable beyond immediate context
- Connect to broader methodology

**Bad learning patterns**:
- Generic advice ("tests are important")
- Obvious statements without insight
- Context-specific without generalization
- Vague principles without examples
